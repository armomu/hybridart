# 2026-04-15

## Flutter Android Gradle 构建修复

**问题**：`dev.flutter.flutter-gradle-plugin` 找不到，构建失败。

**原因**：`android/settings.gradle` 使用旧版写法（无 `pluginManagement` 块），但 `app/build.gradle` 使用新版插件 DSL，两者不兼容。

**修复内容**：
1. `android/settings.gradle`：改为包含 `pluginManagement` + `includeBuild("$flutterSdkPath/packages/flutter_tools/gradle")` 的新版写法，同时声明 AGP 7.3.0 和 Kotlin 1.9.0 插件版本，repositories 保留阿里云镜像 + `http://download.flutter.io`。
2. `android/build.gradle`：移除 `buildscript` 块（插件版本已在 settings.gradle 统一管理），保留 `allprojects.repositories`（含阿里云镜像）。
3. `android/app/build.gradle`：将 `kotlin-android` 改为 `org.jetbrains.kotlin.android` 完整 ID。

## 网络仓库修复（解决 dl.google.com 被墙）

**问题**：Gradle 访问 `dl.google.com`（google()）、`repo.maven.apache.org`（mavenCentral()）时 TLS 握手失败。

**修复**：`settings.gradle` 和 `build.gradle` 中移除 `google()`、`mavenCentral()`、`jcenter()`、`gradlePluginPortal()`，全部替换为阿里云镜像：
- `https://maven.aliyun.com/repository/google`
- `https://maven.aliyun.com/repository/public`
- `https://maven.aliyun.com/repository/gradle-plugin`（仅 settings.gradle）

## 深层修复：Flutter SDK 内置 Gradle 插件仓库

**根因**：Flutter SDK 内部的 `packages/flutter_tools/gradle/settings.gradle.kts`（included build）使用 `FAIL_ON_PROJECT_REPOS` 模式，强制绕过了项目级仓库配置，直接用 `google()` + `mavenCentral()` 下载 AGP。

**修复**：
1. `flutter/packages/flutter_tools/gradle/settings.gradle.kts`：`google()` `mavenCentral()` → `https://maven.aliyun.com/repository/google` + `https://maven.aliyun.com/repository/public`
2. `flutter/packages/flutter_tools/gradle/resolve_dependencies.gradle`：同样替换为阿里云镜像

**flutter.sdk 路径**：`C:\Users\qq894\flutter`（已在 local.properties 配置）

## Gradle Wrapper 升级

**问题**：AGP 7.3.0 要求 Gradle ≥ 7.4，但 `gradle-wrapper.properties` 指定的是 7.2。

**修复**：`gradle-wrapper.properties` 中 `distributionUrl` 由 `gradle-7.2-bin.zip` 改为 `gradle-7.6.4-all.zip`。
